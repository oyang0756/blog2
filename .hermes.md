# Blog

个人博客,Astro 构建。

## 构建与运行

- `npm install` —— 安装依赖
- `npm run dev` —— 本地预览 → http://localhost:4321
- `npm run build` —— 构建静态站 → `dist/`
- `npm run preview` —— 预览构建产物
- `npm run new "标题"` —— 新建草稿文章(自动生成日期+slug)

## 文章管理规范

### 命名与位置

- 所有文章统一放 `src/content/blog/`
- 文件名:`YYYY-MM-DD-slug.md`,如 `2026-07-02-hello-world.md`
- slug 用英文小写 + 连字符(中文标题保留中文也行,见下方 slug 生成脚本)
- 新建文章用 `npm run new "标题"`,**不要手动复制粘贴文件**

### Frontmatter 必填字段

```yaml
---
title: 文章标题            # 必填,≤ 80 字
description: 简短摘要       # 必填,≤ 200 字,用于 SEO 和卡片预览
date: 2026-07-02           # 必填,发布日期
tags: [标签1, 标签2]       # 可选,数组
categories: [分类]          # 可选,数组
cover: /images/...         # 可选,封面图
coverAlt: 描述              # 可选,封面图 alt 文本
draft: false               # 默认 false,草稿期间设为 true
updated: 2026-07-05        # 可选,最后修改日期
---
```

**frontmatter 错误会被 Astro 构建时拦截**(类型 schema 校验)。

### 图片

- 放 `public/images/posts/YYYY-MM-DD/`
- 文章里这样引用:`![alt](/images/posts/YYYY-MM-DD/xxx.png)`
- 单张图 < 500 KB,大图用 WebP

### 标签 vs 分类

- **tags**:多、平级、随性(`[React, 教程, 入门]`)
- **categories**:少、层级、严格(`[前端]` 或 `[生活/旅行]`)
- 分类总数不要超过 7 个,标签随便

### 编辑流程

1. `npm run dev` 起本地预览,改完自动热更新
2. 写完把 `draft: false`,确认 `<title>` 和 `<description>` 已填
3. `git add . && git commit -m "post: 标题" && git push`
4. 接 CI / Vercel / GitHub Pages 后,推送即发布

## 约定

- **文章内容用中文时,frontmatter 字段值用单引号包**(避免 YAML 转义)
- **不引入额外 JS 框架**,除非有强需求
- **重要改动先在 git 分支试**
- **所有源文件以 UTF-8 (无 BOM) 保存**
